Tutorial

  • Open up your Chromebook, and go to chrome://version in the browser.
  • Look at the line that says "Platform:". Note down the last word on there. That's your firmware.
  • Next, download the modified shim file that corresponds to your firmware version.
  • If your board has (unsupported) next to it, it means there is no sh1mmer payload for it. Ping @CoolElectronics#4683 if you need a payload or make one yourself (instructions below)
  • If it has (?) next to it that means it's untested. It may work or it may not
  • Flash the file to a USB. This can be done with the Chromebook Recovery Utility, on Windows with Rufus, or done on Linux with DD. If you're using Rufus, make sure to use DD image mode.
  • Press ESCAPE, REFRESH, and POWER on your Chromebook, press CTRL + D, then enter.
  • The screen will say developer mode is blocked. THIS IS SUPPOSED TO HAPPEN
  • Press ESCAPE, REFRESH, and POWER again
  • Plug in the USB you created
  • Now, wait for a screen with the text "SH1MMER EXPLOIT" to pop up.
  • Press D to deprovision the device and hit enter
  • Once it's done, press E to exit and hit enter. Your chromebook will reboot
  • On the OS Verification screen, press ENTER or SPACE to disable developer mode.

  • If you want to reprovision your device, you can take the same steps, but press R instead of D

Making your own modified shim: ADVANCED USERS ONLY

YOU MUST BE ON LINUX! Windows is not supported
  • Download a raw shim image.
  • Use DD to burn the shim to a usb
  • Plug the shim into a booted unenrolled chromebook
  • Run make_dev_ssd.sh -i /dev/sda --remove_rootfs_verification
  • Unplug it and back into a linux computer
  • DD partition 3 of the usb to a temporary file
  • Do step 1 again
  • Burn the temp file back to partition 3
  • Mount ROOT-A (partition 3 of the usb)
